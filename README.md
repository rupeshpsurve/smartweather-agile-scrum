# SmartWeather Analytics & Agile Scrum Delivery
### Weather Data Analytics | Agile Scrum | Jira | Power BI | Sprint Delivery

---

## 📌 Project Overview

This project involved designing and delivering a **centralised weather analytics 
platform** that aggregates data from multiple weather providers to generate 
actionable business insights for industries including tourism, healthcare, and retail.

A secondary objective was implementing a full **Agile Scrum delivery framework** 
using Jira — covering epics, user stories, sprint planning, backlog management, 
and end-to-end sprint execution.

---

## 🎯 Objectives

- Design a scalable weather analytics platform aggregating multi-source weather data
- Identify key industry use cases across tourism, healthcare, and retail
- Implement a full Agile Scrum framework in Jira for project delivery
- Manage one complete end-to-end sprint lifecycle
- Define KPIs and dashboard requirements for weather-driven business insights
- Establish a repeatable Agile delivery process for future project phases

## 🔄 My Approach (As Business Analyst & Scrum Practitioner)

**1. Requirements & Use Case Discovery**
- Conducted stakeholder discussions and business analysis workshops
- Identified key industry use cases for weather intelligence:
  - Tourism: seasonal demand forecasting
  - Healthcare: weather-related health risk indicators
  - Retail: weather-driven inventory and promotion planning
- Mapped data aggregation requirements across multiple weather API sources

**2. Agile Scrum Framework Setup in Jira**
- Designed and configured complete Scrum workflow in Jira:
  - Created Epics, User Stories with acceptance criteria
  - Configured Scrum board with To Do / In Progress / Done columns
  - Set up backlog with prioritised user stories
  - Created sprint board and release roadmap
- Structured delivery across 3 sprints

**3. Sprint Execution**
- Managed one complete sprint lifecycle end-to-end:
  - Sprint planning — story point estimation, sprint goal definition
  - Daily stand-ups — progress tracking, blocker identification
  - Sprint review — demo of completed stories to stakeholders
  - Sprint retrospective — team improvements identified
- Tracked velocity and sprint burndown

**4. Analytics & Dashboard Design**
- Developed analytics workflows for weather pattern processing
- Defined business-focused KPIs and dashboard requirements
- Designed reporting for: seasonal demand forecasting, health risk indicators, retail trend analysis
